2. Optimisation SEO

La meta description

La meta description d'une page web est une balise meta qui sert à l'indexation de cette page par les moteurs de recherche et les annuaires. Elle doit contenir une description concise du contenu de la page en une ou deux phrases.

Le fichier site map

Sitemaps est un protocole permettant à un webmaster d'informer les moteurs de recherche des adresses d'un site web disponibles pour l'indexation automatique.
Il s'agit d'un plan de site (« sitemap ») compréhensible par les robots d'indexation, rédigé sous forme d'un fichier XML ou texte qui répertorie les URL d'un site permettant ainsi d'inclure des informations complémentaires sur chaque adresse, comme sa date de dernière modification, la fréquence de mise à jour et son importance par rapport aux autres adresses du site.
Cela permet donc aux moteurs de recherche d'explorer le site plus intelligemment pour finnalement permettre un meilleur référencement.

Compresser les images
Sur la version initial, les images étaient dans un format trop lourd (BMP par exemple), trop grande ou d'une résolution trop élevée. On utilise des logiciels comme gtmetrix qui nous permet de télécharger une image plus optimisée.
Réduire le poids d'une image est crédité auprès du BOT de référencement. De plus, la page se chargera plus rapidement donc c'est aussi plus agréable pour les utilisateurs.

3. Optimisation Accessibilité

Définition de l'accessibilité

On parle d'accessibilité lorsque l'on parle de la prise en main du site web, on cherche à rendre celui-ci "user-friendly". Beaucoup de normes doivent être respectées comme le responsive design ou l'accessibilité tous types d'utilisateurs.
Plusieurs points on été corrigé pour rendre le site bien plus accessible.

W3C Validator HTML
Le test W3C HTML effectue une vérification du code et énonce les problèmes qui ne respectent pas le "markup validity". Celui-ci s'assurera que le code HTML soit correctement agencé, ce qui assurera une meilleure expérience aux utilisateurs.
W3C Validator CSS
Le test W3C CSS sera utilisé comme son conjoint, il repère les potientels erreurs CSS tel que la duplication de classe par exemple.
Les contrastes
On remarque qu'il y a à plusieurs endroit un très faible contraste de couleur entre le texte et son background. Il faut penser à tout type d'utilisateur qui pourrait venir sur notre site, notamment toute personne mal-voyante par exemple. Dès standars de contraste peuvent être appliqué pour qu'un plus grand nombre d'utilisateur puissent avoir une expérience agréable sur notre site. Il faut respecter un ratio minimum de 3:1 (source Wave) entre un texte et son background.
On utilise contrast-finder pour calculer un contraste suffisant. Si l'on test un contraste et que celui-ci ne respecte pas le contraste minimum, l'outil suggère plusieurs couleurs alternatives au contraste correct.
Responsive
Certainement l'aspect le plus important d'un site web : le responsive.
S'assurer que celui-ci est adaptaif peut importe l'environnemet est primordial.
Beaucoup de problème on dûe être modifier afin de le rendre totalement responsive.
Tester directement son site sur beaucoup d'environnements est le meilleur moyen d'assurer un responsive parfait.
